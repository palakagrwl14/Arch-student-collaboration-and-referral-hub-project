Project Report: Arch

Student Collaboration & Referral Hub Platform

Overview: Arch is a premium, full-stack web application designed to bridge the gap between college students and professional alumni. The platform empowers students to find teammates, collaborate on projects, and secure job/internship referrals, while providing alumni with a streamlined system to mentor students, share vacancies, and manage candidate applications.

1. Executive Summary

In academic and early-career environments, students often face two key challenges: finding qualified peers to build projects (Collaboration) and obtaining professional career entry points (Referrals). Arch solves both by creating a unified network:

  • The Collaboration Hub: A workspace where students pitch ideas, recruit teammates, and manage project development using native project management tools (task boards, milestones, and shared notes).
  • The Referral Hub: A marketplace where verified alumni post referral slots, review applicant profiles, download resumes, and refer candidates.

2. Key Modules & Features

A. Authentication & Session Management

  • Dual-Role Onboarding: Users register either as a Student or Alumni/Employee.
  • Google Single Sign-On (SSO): Integration of Google OAuth 2.0. The platform dynamically checks if a Google login corresponds to a new user; if so, it prompts them to select their role and sets up their profile seamlessly.
  • Email Verification & Security OTPs: Nodemailer-driven 6-digit OTP verification wizard during registration and email change flows.
  • Secure Session Handshake: Stateless JWT (JSON Web Token) authentication stored securely in client browser storage, attached to API calls via Axios interceptors.

B. The Collaboration Hub

  • Project Feed: A searchable feed of student-led projects, filtered by skills required, open roles, and status.
  • Workspaces: Once a team application is accepted, members gain access to a project workspace containing task boards, milestones, and shared notes.
  • Student Portfolio: Completed projects are verified and showcased directly on students' public profiles.

C. The Referral Hub

  • Referral Vacancy Feed: Searchable listing of job/internship referrals posted by verified alumni. Features parameters like company name, job role, package details, eligibility criteria (CGPA, batches), and remaining referral slots.
  • Applications Manager: Students upload their resume and cover pitch. Alumni can review candidate cards, audit resumes, and update application status (Applied, Under Review, Referred, Rejected).
  • Ratings & Reviews: Students can rate their referral experience with alumni, building accountability.

D. Administrative Panel

  • Alumni Verification Queue: Alumni must upload employment verification proofs. Admins review proofs in a dedicated panel to approve or reject them.
  • Report Auditing: System flag triggers when listings are reported, allowing admins to take down listings or ban users.

3. System Safeguards & Security Fixes

To ensure production-grade security, the following loopholes were addressed:

  • Email Change Verification: The old email remains active in the database until the verification code sent to the new email is correctly entered.
  • SSO Password Setup: Google SSO users do not have a password initially. The Settings panel detects this, hides the "current password" requirement, and changes the field to "Set Account Password," allowing SSO users to define one.
  • Account Deletion Safeguards: Prior to deletion, the system counts the user's active applications and posted referrals and prompts them with a confirmation warning: "You have X active applications and Y posted referrals. Deleting your account will remove them permanently."
  • Verification Gatekeepers: Unverified alumni profiles are blocked from creating new referral postings until approved by an administrator.

5. Database Schema

The database consists of 14 relational tables, designed with foreign key cascades and indexes for performance:

  • users: Stores core profiles (id, email, password_hash, role, college, skills, company, verification_status).
  • projects: Collaboration listings posted by students.
  • team_members: Joins users to projects they are part of.
  • team_applications: Applications submitted by students to join project teams.
  • workspace_tasks: Tasks assigned within a project workspace (Todo/In Progress/Done).
  • workspace_notes: Shared notes within project workspaces.
  • milestones: Key checkpoints inside workspaces.
  • portfolio_entries: Verified accomplishments shown on student profiles.
  • referrals: Vacancy listings posted by alumni.
  • referral_applications: Applications submitted by students for referrals.
  • ratings: Scores (1-5) given to alumni by referred students.
  • reports: Moderation flags for inappropriate listings.
  • notifications: Real-time push alerts for user actions.
  • audit_log: Track of administrative modifications.
